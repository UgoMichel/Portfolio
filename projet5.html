Context

Sujet

Un site “classique” nommé MyAvatar en mode server-side rendering inspiré du service Gravatar

Les missions

Cette application devait à travers une inscription permettre d’associer une adresse mail à une photo de profil. Ainsi, à partir de cette adresse mail, les différents sites web et applications pouvaient charger l’image de profil correspondante depuis le service MyAvatar. Cela avait pour but d’éviter à la fois aux autres applications de stocker cette image de leur côté mais aussi de demander à l’utilisateur d’uploader sa photo de profil.

Par conséquent, l’objectif était que n’importe quelle application manipulant l’adresse mail d’un utilisateur pouvait facilement charger une image depuis MyAvatar.

Base de Données

Aucune base de données nous était mise à notre disposition au début du projet. Nous étions libres d'utiliser du MySQL ou bien du PostgreSQL.

Mesure & Optimisation

En ce qui concerne les mesures, nous avons dû appliquer et réutiliser les notions que l’on a pu voir lors des cours de Symfony. On peut donc penser à l’installation d’un projet Symfony, à l’utilisation des migrations, des entités et des formulaires sous le framework web. De plus, nous étions totalement libres au niveau du frontend, nous devions bien évidemment utiliser twig pour l’interface. C’est pourquoi en plus de twig, nous avons choisi d'utiliser le framework le plus optimisé pour les projets web, nommé Tailwind CSS.

Composants à intégrer

Les composants obligatoires à intégrer dans ce projet étaient de réaliser un site qui permet de s’inscrire en précisant un login, un mot de passe, une adresse email et surtout une photo de profil. Puis, il fallait que l’utilisateur puisse modifier les informations de son compte, notamment son adresse email et sa photo de profil. Il devait même pouvoir supprimer son compte tout en gérant en parallèle le fichier contenant la photo de profil. Enfin, on devait avoir accès à l’image de profil à partir du code obtenu en chiffrant l’adresse email de l’utilisateur en MD5.

Langages autorisés

HTML, CSS (framework autorisé), PHP (Symfony).

Analyse

Lors de ce projet, j’ai pu développer et apprendre de nouvelles compétences techniques en informatique.

Côté base de données

En ce qui concerne la base de données, nous avons choisi avec tous les membres du groupe de partir sur une base de données MySQL comme lors des cours de Symfony. Ainsi, j’ai pu appliquer ce que j’avais pu voir en cours comme la configuration de la base de données dans le projet Symfony. Puis, grâce au système de migration de Symfony, j’ai pu créer les différentes tables nécessaires.

Côté PHP

Pour le PHP, c’est la première fois que j’utilisais un framework PHP au sein d’un projet web. Ainsi, j’ai pu appliquer tous les principes de base de Symfony vue en cours. Cela m’a permis de réellement comprendre comment installer et configurer le framework au sein d’un projet. Puis, j’ai pu utiliser les différents outils qu'offre Symfony comme son système de migration, son système de routage, mais aussi sa façon particulière de traiter les formulaires. Enfin, ce projet m’a été utile pour solidifier mes compétences en développement backend.

Côté HTML & CSS

En ce qui concerne le frontend, nous avons choisi d’utiliser le framework Tailwind CSS. Nous l’avons préféré à Bootstrap pour des raisons d’optimisation. De plus, je suis amené à l’utiliser également lors de mes périodes en entreprise. Par conséquent, j’ai dans un premier temps utilisé les connaissances que je possédais afin de réaliser la meilleure application possible. Puis dans un second temps, ce projet m’a permis d’apprendre davantage vis à vis de ce framework. Ainsi, j’ai pu acquérir de nouvelles compétences en frontend.

Amelioration

Pistes d’amélioration

Pour les pistes d’amélioration, nous pouvons bien évidemment ajouter de nouvelles fonctionnalités utilisateurs. Cependant, ce projet doit rester simple lors de son utilisation. Ainsi, on peut penser à l’ajout de plusieurs options de personnalisation au niveau de la page de profil par exemple.

Enfin, une version mobile est toujours la bienvenue afin que ce service soit accessible partout, mais aussi par d'avantages d’utilisateur.
